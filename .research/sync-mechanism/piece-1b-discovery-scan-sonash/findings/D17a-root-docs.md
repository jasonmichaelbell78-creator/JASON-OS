## Learnings for Methodology

1. **llms.txt is a key discovery artifact.** Provides a pre-indexed view of all skills (77 listed by name + purpose snippet), faster than scanning .claude/skills/ directories. Future scans should read llms.txt first as quick-reference before diving into SKILL.md files.

2. **ROADMAP.md.bak portability classification edge case.** `.bak` files need a classification rule. Proposed: any `.bak` = `status: archived`, `portability: not-portable`, deletion candidate.

3. **LICENSE vs README contradiction.** `LICENSE` file is Apache 2.0 but README.md footer says "Proprietary — All rights reserved." README v2.2 from 2026-02-23 predates the Apache license addition. Flag for D21 as cross-document inconsistency.

4. **META_ROADMAP is a portable concept, not just a portable doc.** The cross-plan routing pattern (lanes, sequenced execution, at-a-glance inventory) should be adopted by JASON-OS independently of the SoNash content. This belongs in "patterns to extract" rather than just "portable files."

5. **DOCUMENTATION_INDEX.md generator** is more valuable than the generated file. The D-agents covering scripts/docs/ should flag the generator script as priority portable. JASON-OS has no equivalent doc cataloger — low priority at 19 root files, but important at scale.

6. **CLAUDE.md gate annotation system is a JASON-OS improvement.** SoNash CLAUDE.md v6.0 has bare rule text. JASON-OS CLAUDE.md v0.1 added GATE/BEHAVIORAL/MIXED/NEEDS_GATE annotations. This improvement should NOT be reverted when syncing.

7. **SESSION_CONTEXT.md session counter is fully honor-system.** Tracked in SESSION_CONTEXT.md itself (Session #287). No tooling enforces it. Purely behavioral.

8. **Auto-generated files still need `status: active`.** llms.txt and DOCUMENTATION_INDEX.md are active documents even though machine-produced. Distinction: `portability: sanitize-then-portable` (for llms.txt pattern/generator) vs `not-portable` (for DOCUMENTATION_INDEX content tied to SoNash doc tree).

9. **File size distribution.** DOCUMENTATION_INDEX.md (7020 lines) dominates. For files above 1000 lines, headings + first 100 + last 50 is sufficient for classification. Full reads needed only for CLAUDE.md and SESSION_CONTEXT.md schema extraction.
